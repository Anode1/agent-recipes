Authentication Rules (Global)
===============================================================================

Protected endpoints require a valid token via:
- Authorization: Bearer <token> (preferred)
- OR ?token=<token> (GET only - fallback)

If token is missing:
    401 {"error":"missing_authentication_token"}

If token invalid:
    401 {"error":"invalid_or_expired_token"}

If token references nonexistent user:
    401 {"error":"user_not_found"}

Public endpoints:
    api/status
    api/login
    api/register
    api/password_reset/*
    api/<static assets>


Protected endpoints:
    api/users/{id}
    api/users/send_registration
    api/admin_users/*
    ...

Registration - send
===============================================================================
Status: IMPLEMENTED

Path:
    POST api/users/send_registration

Client:
    Authorization: Bearer <token>
    {
      "email": "someone@example.com",
      "username": "Some Name",
      "role": "EDITOR"          // optional: USER (default) | EDITOR | ADMIN
    }

Server:
    200 {
        "status": "sent",
        "created": true,             // whether a new USER row was created
        "role_ensured": true         // whether the requested role was added (for ADMIN/EDITOR invites)
    }

Errors:
    400 {"error":"email_required"}
    400 {"error":"unknown_email"}             // email not in USER and no username provided to auto-create
    400 {"error":"invalid_role"}              // role not in {USER, EDITOR, ADMIN}
    401 {"error":"missing_authentication_token"}
    401 {"error":"invalid_or_expired_token"}
    401 {"error":"user_not_found"}
    403 {"error":"admin_role_required"}       // caller must have ADMIN role to invite ADMIN or EDITOR
    409 {"error":"already_registered"}        // email already has an active password

Notes:
- registration_token stored in USER.
- EDITOR callers may invite USER roles only; only ADMIN may invite ADMIN or EDITOR.

Admin_Users - change role
===============================================================================
Status: IMPLEMENTED

Path:
    POST api/admin_users/{user_id}/role

Client:
    POST api/admin_users/7/role
    Authorization: Bearer <token>
    Content-Type: application/json
    { "role": "ADMIN" }

Server:
    200 {
        "status": "updated",
        "user_id": 7,
        "role": "ADMIN"
    }

Errors:
    400 {"error":"..."}                   (missing / invalid JSON body)
    401 {"error":"..."}                   (no / invalid token)
    401 {"error":"unauthorized"}          (caller is not EDITOR or ADMIN)
    403 {"error":"admin_role_required"}   (caller is authorized but NOT ADMIN)
    400 {"error":"invalid_user_id"}       (non-numeric {user_id})
    400 {"error":"cannot_modify_self"}    (caller == target)
    400 {"error":"invalid_role"}          (role not ADMIN or EDITOR)
    404 {"error":"user_not_found"}        (no such target user)
    409 {"error":"last_admin"}            (demoting the only remaining active ADMIN)

Notes:
- ADMIN-only: passes requireAuthorized (EDITOR or ADMIN) AND an explicit
  user.isAdmin() check; non-admin authorized users get 403 admin_role_required.
- Single-role replace via UserDAO.setRole. Body field "role" is trimmed/upper-cased,
  accepted only as "ADMIN" or "EDITOR" ("USER" is rejected).
- Self-modification is blocked; cannot demote the last active admin.
